## 본문
마이크로소프트가 지난 12월부터 자사 개발자 수천 명에게 개방했던 Anthropic의 AI 코딩 도구 **Claude Code**의 라이선스를 대거 회수하기 시작. 6개월간 사내에서 폭발적 인기를 끌었으나, 오히려 자사 **GitHub Copilot CLI**의 입지를 위협한 것이 부메랑으로 돌아오다.  
  
#### 주요 내용  
  
- **대상 조직**: Windows, Microsoft 365, Outlook, Teams, Surface 등을 담당하는 **Experiences + Devices(E+D)** 팀이 6월 말까지 Claude Code 사용을 단계적으로 중단하다.  
- **전환 도구**: 엔지니어들은 향후 몇 주 안에 워크플로를 **GitHub Copilot CLI**로 이전하도록 권고받다.  
- **명분과 실리**: 마이크로소프트는 "에이전틱 CLI 도구를 Copilot CLI로 일원화"한다는 명분을 내세웠으나, **6월 30일 회계연도 종료일**에 맞춘 비용 절감 조치라는 분석이 제기되다.  
- **경영진 입장**: E+D 부문 부사장 **Rajesh Jha**는 내부 메모에서 "Claude Code는 중요한 학습 자산이었지만, Copilot CLI는 마이크로소프트의 리포지토리·워크플로·보안 요구에 직접 맞춰 형성할 수 있는 제품"이라고 밝히다.  
  
#### 배경과 파장  
  
- 마이크로소프트는 코딩 비경험자인 디자이너·PM에게도 Claude Code로 프로토타이핑을 장려해왔으며, 사내 개발자들은 Copilot CLI보다 **Claude Code를 선호**해온 것으로 드러나다.  
- 양 제품 간 격차를 좁히기 위해 마이크로소프트는 한때 **Cursor 인수**를 검토했으나, 규제 우려를 피해 다른 AI 스타트업으로 눈을 돌리다.  
- 다만 **Anthropic과의 파트너십은 유지**되다. Claude Sonnet 4.5·Opus 4.1·Haiku 4.5는 여전히 Microsoft Foundry와 Microsoft 365 Copilot에서 활용되며, Copilot CLI 내부에서도 Anthropic 모델을 호출할 수 있다.  
- 작년 기준 마이크로소프트 엔지니어링 팀의 91%가 GitHub Copilot을 사용했으나, 최근 6개월간 Claude Code 확산으로 그 수치가 흔들렸다는 점이 이번 조치의 핵심 동인으로 지목되다.  
  
#### 전망  
  
공은 이제 **GitHub 팀**으로 넘어가다. Copilot CLI가 Claude Code를 능가하는 수준의 에이전틱 코딩 경험을 제공할 수 있는지가 향후 수개월간 마이크로소프트 내부 AI 전략의 시금석이 될 전망이다.
